1. Descriere Proiect

Pe baza datelor existente si a observatiilor semnalate de utilizatori (autentificati via un cont existent pe o platforma Web cu caracter social precum Facebook ori Twitter), se doreste realizarea unei harti interactive -- via un serviciu public de cartografiere precum OpenStreetMap -- referitoare la traseele migratiei persoanelor la nivel de natiune, de continent sau de glob. Raportarea unui eveniment migratoriu se va putea efectua fie direct in cadrul aplicatiei Web, fie folosind termenul de continut '#migration' in cadrul retelei sociale preferate de utilizator. Sistemul propus va fi capabil sa figureze si sa ofere statistici vizand aceste migratii sezoniere in functie de zona geografica (e.g., tranzit pe teritoriul unei tari), perioada de timp, climat, consideratii demografice etc. Fiecare nou raport introdus de utilizator va fi semnalat si in cadrul unui flux de stiri Atom. Sistemul Web dezvoltat va oferi statistici relevante disponibile ca documente HTML, CSV, JSON si PDF, putand fi partajate la nivelul retelelor sociale. Folosind datele disponibile, se vor putea oferi predictii referitoare la migrarea persoanelor spre un areal geografic.

4. Progresul inregistrat

Modelarea paginilor principale (use-case)

Un utilizator al aplicatiei HuWr va putea sa se logheze (utilizand contul din aplicatie, sau utilizand Twitter) avand astfel o serie de facilitati precum adaugarea de migratii, vizualizarea de statistici sau predictii si de asemenea de a consulta harta interactiva. Un utilizator care nu este logat va fi limitat doar la vizualizarea migratiilor pe o harta interactiva si de a consulta o lista de migratii. De asemenea, toti utilizatorii vor putea accesa API-ul pus la dispozitie de aplicatia noastra si de asemenea cateva detalii legate de noi (dezvoltatorii aplicatiei).

5. Tehnologii utilizate

PHP

PHP este un limbaj de programare ce se foloseste inglobat in codul HTML pentru a oferi diferite functionalitati la nivel de server (back-end). Am ales acest limbaj deoarece exista framework-uri mature pentru utilizarea acestuia, este simplu de utilizat si comunitatea ofera multa documentatie pe internet. De asemenea, pentru PHP respectiv Laravel exista foarte multe API-uri care ne-au ajutat sa oferim utilizatorilor nostri o experienta cat mai placuta.

Laravel framework

Laravel este un framework open-source bazat pe PHP si pe arhitectura MVC(model-view-controller). Acest framework ne-a permis sa modularizam aplicatia si ne-a oferit ajutor in crearea unei ierarhii de fisiere cat mai usor de utilizat, reutilizat si intuitive. Un plus pentru care am ales acest limbaj, asa cum am mentionat si mai devreme este faptul ca exista foarte multe pachete care ne-au ajutat sa gestionam mai bine bazele de date, sa reutilizam si sa modularizam view-urile, etc.

HTML/CSS/Javascript

HTML/CSS/Javascript ne-au ajutat sa oferim o experienta cat mai placuta utilizatorilor la nivel de client utilizand HTML pentru a marca elementele ce tin de interfata cu care va interactiona utilizatorul, apoi stilizand aceste elemente utilizand foi de stiluri CSS urmand apoi sa oferim diferite functionalitati din Javascript fara a fi nevoie sa facem apeluri din nou la server.

Baze de date MySQL

Pentru a stoca datele necesare aplicatiei am preferat sa folosim o metoda de stocare bazata pe baze de date relationale utilizand MySQl, deoarece este una din cele mai simplu de folosit baze de date, oferind in acelasi timp functionalitati decente pentru cerintele aplicatiei noastre.

AJAX

AJAX este o metoda de dezvoltare a aplicatiilor la nivel de client ce permite trimiterea de cereri la server intr-un mod asincron, iar apoi utilizand Javascript realizand schimbarile corespunzatoare la nivel de interfata direct din DOM, fara a fi nevoie sa reincarcam pagina (obtinand astfel un avantaj considerabil de performanta). Am folosit aceasta metoda de dezvoltare pentru a oferi o experienta cat mai "smooth" utilizatorilor, pentru a oferi mesaje de confirmare sau pentru incarca diferite date de la server fara a reincarca pagina, astfel dandu-i utilizatorului impresia unei aplicatii foarte "smooth".

6. Servicii utilizate

Blade

Blade este un plugin al framework-ului Laravel ce permite sectionarea view-urilor si crearea de template-uri pentru acestea astfel incat sa reutilizam diferite componente din view-uri in cat mai multe pagini evitand codul duplicat si favorizand modularizarea.

Eloquent

Eloquent este un alt plugin al framework-ului Laravel si nu este decat un ORM(object-relational mapping) care ne-a ajutat sa mapam modelele create in cadrul aplicatiei noastre cu datele din baza de date astfel incat sa ne fie cat mai simplu sa accesam datele necesare pentru experienta utilizatorului.

D3.js

D3.js este o biblioteca Javascript care se bazeaza pe crearea de reprezentari grafice bazate pe date. In acest sens, am utilziat aceasta biblioteca pentru a crea diferite statistici despre migratii la nivel de tara(ex: statistici referitoare la motivul migratiei, statistici bazate pe anui sau consideratii demografice)

Swagger-doc

Swagger este un framework ce ajuta dezvoltatorii de API-uri sa isi documenteze adecvat serviciul pus la dispozitie altor dezvoltatori. Avand un API minimal ce ofera modalitati de consulta migratii, de a consulta predictiile si statisticile puse la dispozitie de aplicatia noastra, am avut nevoie de o metoda de documentare a cererilor intr-un mod standardizat si pentru acest lucru am preferat Swagger.

Open-Street-Map(Leaflet + diferite plugin-uri)

Open-Street-Map este un serviciu de cartografiere la nivel de web, iar Leaflet reprezinta o biblioteca de Javascript peste acest serviciu, ce ne-a oferit impreuna cu diferitele plugin-uri open-source sa realizam o harta interactiva responsive care ofera utilizatorului o experienta placuta.

Twitter API - impreuna cu Thujohn API

Twitter este o platforma de social media ce le permite utilizatorilor aplicatiei sa impartaseasca migratiile adaugate din cadrul aplicatiei sau statisticile generate de aplicatia noastra cu prietenii lor. De asemenea, procesul poate fi si invers, utilizatorii de Twitter pot adauga migratii direct printr-o postare chiar si fara a avea un cont in cadrul aplicatiei.

Scheduler

Pentru a mentine aplicatia cu date cat mai relevante si cat mai recente am simtit nevoia de a folosi un Scheduler ce ne permite ruland o comanda sa putem prelua ultimele date de pe Twitter si sa le adaugam in baza de date la diferite intervale de timp.

Bootstrap + FontAwesome

Bootstrap si FontAwesome sunt biblioteci CSS ce ajut la stilizarea elementelor HTML intr-un mod cat mai familiar si atractiv pentru utilizatori si de asemenea asigura ca elementele stilizate utilizand aceste librarii sunt responsive si eficient facute astfel incat sa oferim utilizatorilor o experienta frumoasa.

MrRio/jsPDF

Pentru a exporta statisticile generate utilizand libraria D3 in formatul PDF a fost nevoie sa folosim un pachet ce ne-a usurat munca si ne-a ajutat apoi si sa impartasim acea statistica si pe Twitter si acest pachet s-a pliat cel mai bine pentru ce aveam noi nevoie.

Roumen/Feed - Stiri ATOM

Toate migratiile adaugate in cadrul aplicatiei apar intr-o pagina separata, insa unii utilizatori s-ar putea sa doreasca sa aiba nevoie de un flux de stiri atom legate de migratii si am oferit acest lucru utilizand standardul existent impreuna cu pachetul specificat mai sus ce ne-a ajutat sa fim mai eficienti.
