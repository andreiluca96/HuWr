1. Descriere Proiect

Pe baza datelor existente si a observatiilor semnalate de utilizatori (autentificati via un cont existent pe o platforma Web cu caracter social precum Facebook ori Twitter), se doreste realizarea unei harti interactive -- via un serviciu public de cartografiere precum OpenStreetMap -- referitoare la traseele migratiei persoanelor la nivel de natiune, de continent sau de glob. Raportarea unui eveniment migratoriu se va putea efectua fie direct in cadrul aplicatiei Web, fie folosind termenul de continut '#migration' in cadrul retelei sociale preferate de utilizator. Sistemul propus va fi capabil sa figureze si sa ofere statistici vizand aceste migratii sezoniere in functie de zona geografica (e.g., tranzit pe teritoriul unei tari), perioada de timp, climat, consideratii demografice etc. Fiecare nou raport introdus de utilizator va fi semnalat si in cadrul unui flux de stiri Atom. Sistemul Web dezvoltat va oferi statistici relevante disponibile ca documente HTML, CSV, JSON si PDF, putand fi partajate la nivelul retelelor sociale. Folosind datele disponibile, se vor putea oferi predictii referitoare la migrarea persoanelor spre un areal geografic.

4. Progresul inregistrat

Modelarea paginilor principale(use-case)

Un utilizator al aplicatiei HuWr va avea la dispozitie sa se logheze(utilizand contul din aplicatie/fie utilizand Twitter) avand astfel o serie de facilitati precum adaugarea de migratii, vizualizarea de statistici sau predictii si de asemenea de a consulta harta interactiva. Un utilizator care nu este logat va fi limitat doar la vizualizarea migratiilor pe o harta interactiva si de a consulta o lista de migratii. 5. Tehnologii utilizate

PHP

PHP este un limbaj de programare ce se foloseste inglobat in codul HTML pentru a oferi diferite functionalitati la nivel de server(back-end). Am ales acest limbaj deoarece exista framework-uri mature pentru utilizarea acestuia, este simplu de utilizat si multa documentatie pe internet. De asemenea, pentru PHP respectiv Laravel exista foarte multe API-uri care ne-au ajutat sa oferim utilizatorilor nostri o experienta cat mai placuta.

Laravel framework

Laravel este un framework open-source bazat pe PHP si pe arhitectura MVC(model-view-controller). Acest framework ne-a permis sa modularizam aplicatia si ne-a oferit ajutor in crearea unei ierarhii de fisiere cat mai usor de utilizat(si de reutilizat) si intuitive. Un plus pentru care am ales acest limbaj, asa cum am mentionat si mai devreme este faptul ca exista foarte multe pachete care ne-au ajutat sa gestionam mai bine bazele de date, sa reutilizam si sa modularizam view-urile, etc.

HTML/CSS/Javascript

HTML/CSS/Javascript ne-au ajutat sa oferim o experienta cat mai placuta utilizatorilor la nivel de client utilizand HTML pentru a marca elementele ce tin de interfata cu care va interactiona utilizatorul, apoi stilizand aceste elemente utilizand foi de stiluri CSS urmand apoi sa oferim diferite functionalitati din Javascript fara a fi nevoie sa facem apeluri din nou la server.

Baze de date MySQL

Pentru a stoca datele necesare aplicatiei am preferat sa folosim o metoda de stocare bazata pe baze de date relationale utilizand MySQl, deoarece este una din cele mai simplu de folosit baze de date, oferind in acelasi timp functionalitati decente pentru ce are nevoie aplicatia noastra.

AJAX

AJAX este o metoda de dezvoltare a aplicatiilor la nivel de client ce permite efectuarea de operatii de tip REST intr-un mod asincron, iar apoi utilizand Javascript realizand schimbarile corespunzatoare la nivel de interfata direct din DOM, fara a fi nevoie sa reincarcam pagina(obtinand astfel un avantaj considerabil de performanta). Am folosit aceasta metoda de dezvoltare pentru a oferi o experienta cat mai "smooth" utilizatorilor pentru a oferi mesaje de confirmare sau pentru incarca diferite date de la server fara a reincarca pagina, astfel dandu-i utilizatorului impresia unei aplicatii foarte "smooth".
